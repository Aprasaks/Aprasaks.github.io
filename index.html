CodeLoungeV2 — 코드 플레이그라운드 개인

JavaScript, React, Vite, Monaco Editor

  • 설명: HTML/CSS/JS 코드를 작성하면 우측에 즉시 미리보기가 반영되는 학습형 코드 플레이그라운드.
  • 구현 방식:
    • monaco-editor로 코드 입력 영역 구현 → 자동완성, 문법 강조 제공.
    • 입력된 HTML/CSS/JS를 조합해 <iframe>에 주입 → 입력 즉시 미리보기 갱신.
    • localStorage로 코드 자동 저장/복원 → 백엔드 없이도 작성 중 데이터 보존.
  • 선택/비선택 이유:
    • 미리보기 구현은 빌드/서버 의존도를 줄이기 위해 iframe을 선택.
    • 상태 저장은 초기 단계에서 DB·인증을 도입하지 않기 위해 localStorage로 대체.
  • 아쉬운 점: 실시간 협업(채팅/동기화)은 미구현. 추후 WebSocket 학습 후 확장 예정.

Q&A 기반 LMS — 팀 프로젝트 팀

React, TypeScript, Vite, Tailwind CSS, Axios, Zustand

  • 개요: 질문·답변·채택 흐름을 중심으로 학습 Q&A를 지원하는 LMS.
  • 내 역할 (내가 한 것만 기재):
    • 공통 컴포넌트: Button·Input 초안은 인라인 스타일로 시작했으나, 팀 회의 후 cn 유틸(클래스 병합)로 전환 → 스타일 일관성과 재사용성 향상.
    • 질문 작성/수정: 외부 마크다운 에디터 라이브러리 없이 직접 입력/툴바/미리보기 구성.
    • 질문/답변 API 연동: 팀 합의로 공통 fetcher(멘토 제공)를 채택해 요청/에러 처리 표준화.
  • 선택/비선택 이유:
    • 스타일 시스템은 Tailwind + cn으로 단순화 → 디자인 토큰과 조합식 스타일을 팀 전체 코드에 일관성을 주기 위해 사용.
    • 마크다운 에디터는 학습을 위해 직접 구현을 선택 → 기존 블로그 프로젝트를 통해 에디터 라이브러리를 여럿 사용했었고 이번엔 한번 구현해보자 라는 생각으로 시도.
    • 데이터 요청은 구현 편차를 줄이기 위해 공통 fetcher 사용 → 코드 중복과 실수 감소.
